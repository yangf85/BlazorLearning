## 代码提交规范

### Commit Message格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type类型
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档修改
- `style`: 代码格式修改
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

### 示例
```
feat(auth): 添加JWT认证功能

- 实现JWT Token生成和验证
- 添加登录API端点
- 集成ASP.NET Core Identity

Closes #123
```

## 问题记录和解决方案

### 常见问题汇总
_在学习过程中遇到的问题和解决方案将记录在这里_

<details>
<summary>使用OpenApi来调试接口</summary>  

**首先把Api项目里面的 launchSettings.json文件中的 "launchBrowser": true,**  

- 使用Scalar 安装Scalar.AspNetCore包
修改launchSettings.json文件,添加一个配置项 "launchUrl": "scalar/v1",

然后添加如下代码
```
if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
    app.MapScalarApiReference(options =>
    {
        options.WithTitle("我的 API 文档")
               .WithTheme(ScalarTheme.BluePlanet) // 或其他主题
               .WithDefaultHttpClient(ScalarTarget.CSharp, ScalarClient.HttpClient);
    });
}

```

- 仍然使用Swagger 安装Swashbuckle.AspNetCore 然后添加如下代码   


```
if (app.Environment.IsDevelopment())
{
    app.MapOpenApi();
    app.UseSwaggerUI(options =>
    {
        options.SwaggerEndpoint("openapi/v1.json", "v1");//设置文档路径
        options.RoutePrefix = "";
    });
}
```
**学习到的知识点**：
通过使用Open Api，可以简化接口调试流程，提升开发效率。
</details>
